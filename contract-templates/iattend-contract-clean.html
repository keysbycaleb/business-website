1. Project Overview

Project Name: iAttend Enrichment Dashboard

Project Description: Provider will design and develop a custom web-based dashboard for Client's after-school enrichment program operations, serving the Lynwood Unified School District (12 school sites). The dashboard will enable attendance tracking, student roster and waitlist management, SEL (Social-Emotional Learning) growth tracking, and reporting capabilities for district partners.

Engagement Model: Project Build with Payment Plan (Client owns intellectual property)

2. Scope of Work & Deliverables

Provider agrees to deliver the following ("Deliverables"):

  • Dropdown selection for all 12 school sites
  • Student roster management per site
  • Waitlist tracking with easy student swap functionality
  • Attendance logging with percentage tracking
  • SEL growth input with percentage tracking
  • Calendar view with historical data filtering (by week, month, school year)
  • Visual dashboard with charts and at-a-glance metrics
  • Secure login for 3 authorized users
  • Clean, polished interface suitable for district partner viewing
  • Comprehensive technical documentation at handoff (approximately 5 hours of documentation work included)

Specifically Excluded from Scope:

  • "Streak" feature (highlighting attendance milestones)
  • Premium animated analytics with advanced visualizations
  • Native mobile app development (iOS/Android apps)
  • Content creation or data entry

10. Infrastructure & Hosting (Firebase)

This project will be built on Google Firebase, a cloud platform that provides database, authentication, and hosting services. Per Section 5, Client will own the Firebase project and be responsible for the associated Google Cloud billing account.

How Firebase Billing Works: Firebase uses consumption-based pricing, meaning costs scale with usage (database reads/writes, bandwidth, etc.) rather than a fixed monthly fee. For an internal tool with 3 users and ~12 school sites, costs are expected to remain within Firebase's free tier or minimal paid tier (typically $0–$25/month).

Cost Scaling: If iAttend expands significantly (e.g., adding many more districts, hundreds of concurrent users, or high-frequency data operations), Firebase costs may increase. Provider will architect the system with cost efficiency in mind, but Client is responsible for monitoring their Firebase billing dashboard and any costs incurred.

Provider's Responsibility: Provider will implement cost-conscious architecture (efficient queries, appropriate caching, optimized data structures) and set up billing alerts in the Google Cloud Console to notify Client if costs exceed expected thresholds.

Client's Responsibility: Client owns the Firebase billing account and is responsible for all infrastructure costs after ownership transfer. Provider is not liable for Firebase costs, including unexpected spikes due to usage growth, inefficient third-party integrations added after handoff, or external factors.
